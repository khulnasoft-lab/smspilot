File Structure

Before installing Zender, please check if all the important directories and files are present in the "Install" directory on your downloaded zip from codecanyon. We want to make sure that you'll be able to install Zender without any problems.

  • system Contains the important core files of the system
    • configurations Contains the environment configurations
    • controllers Contains the app controllers
    • models Contains the database models
    • plugins Contains the framework plugins and custom libraries
    • storage Temporary cache data are stored here
    • framework.php The core file of the framework
  • templates Contains the front-end template files
    • _apidoc Contains API documentaion template files (We are using apidocjs)
    • _assets Contains the overall frontend libraries (We hate repeating files)
    • _install Contains the installer template files (Must be deleted after installation)
    • _mail Contains the email template files
    • _scss Contains the scss stylesheets for theme customization
    • dashboard Contains the dashboard template files
    • default Contains the landing page template files
  • uploads Contains system uploads
    • avatars Contains user avatar images
    • builder Contains the app builder files
    • sheets Contains the import excel files
    • themes Contains the theme uploaded files
  • vendor Contains the composer dependencies
  • .htaccess Mod rewrite rules
  • .nginx.conf Nginx rewrite rules
  • index.php Request handler of the framework
  • install.sql Installer SQL file

Installing

Requirements

Please make sure that your server meets the following server & extension requirements.

  • Apache >= 2.4 or Nginx >= 1.17
  • mod_rewrite module if using apache
  • PHP >= 7.1
  • MySQL >= 5.6
  • php-gd
  • php-mbstring
  • php-intl
  • php-zip
  • php-curl
  • php-xmlreader
  • php-bcmath

Uploading

Download an ftp client to upload the files to your server. We recommend FileZilla for this.

If you have experience in linux, it's faster to upload the files using rsync.

Unzip the file you downloaded from codecanyon, you'll see 4 directories inside it. You will only need to upload the all contents of the "Install" directory.

Zender can be installed in the root directory or a sub-directory.

NGINX Rules

If your server is nginx, please include the .nginx.conf to your server configuration block. This is how nginx works.

Use this syntax

include /path/to/zender/.nginx.conf;

Permissions

After uploading the files to your server, you should make sure that the required files has the permissions needed. The following directory and files must be writable, please chmod them to the required permissions.

  • system Contains the important core files of the system
    • configurations
      • cc_env.inc 775 dont worry, this file is not accessible to public
      • cc_ver.inc 775 dont worry, this file is not accessible to public
    • storage 775 including sub-directories
  • uploads 775 including sub-directories

Changing permissions of files can be done using FileZilla or cPanel file manager.

Database

To install zender, you will need a database. If you haven't created yet, go to your cPanel and create a new database.

Set the database collation to utf8mb4_unicode_ci for better character support.

If you don't have enough knowlegde on how to create a database on your server, please contact us using the contact form on our codecanyon profile here.

Install

After uploading, changing of permissions and creating a mysql database, open your site url in the browser and you should be redirected to the install wizard:

Installation Steps

  • Enter your site name, we recommend this to be short from 3 to 10 characters with no spaces.
  • Select your site protocol, choose HTTPS if your server has an ssl certificate and https compliant otherwise select HTTP. It is important that your properly set this to prevent issues connecting to our server.
  • Enter your site description, be descriptive to boost your site's SEO effectivity.
  • Enter your database hostname:port, database name, database username and database user password
  • Enter your full name, preffered admin username and password. The first user is automatically set as super admin and cannot be deleted from system. User roles is already planned to be added on the next update, please view the zender official roadmap on the community forums for more info.
  • When you're done filling up the required fields, click the Install Zender button.
  • It should take less than a minute to install the database and system data. After finishing the installation. You will see something like the image below, follow the further instructions and you're done!

If you're not able to install zender successfully on your server, please contact us using the contact form on our codecanyon profile here. We will assist you for free.

Message Splitting

Zender gateway app supports sending long messages, it will split to multiple parts if the receiving device doesn't support long messages.

Smart Sending

If the user didn't choose a device before sending a message, the system will automatically decide what to use based on the amount of pending messages of devices, the device with lowest pending messages will be used to send the message. This feature also applies to API sending.

Android KitKat

Dual sim sending for this android version is not supported, the sdk version has no in-built function for this purpose and we can't do anything about it. If the user choose to send a message on sim 2 on a KitKat device, it will be sent using sim 1.

Widgets

Zender navigation system is so smart that it only needs to refresh the whole page when logging-in, logging-out and registering and because of that, zender uses modals, these modals dynamically loads contents and saves a lot of bandwidth and resources.
For advertisements, you can choose the block type widget, these blocks can be easily added to template files by just including a one line block code.

After adding a modal, you can easily copy the toggle attribute by clicking the copy button of the row. The modal toggle attribute looks like this:

zender-toggle="c20ad4d76fe97759aa27a0c99bff6710"

To make the modal work, just add the toggle attribute to any links or images like this:

HTML link example

<a href="#" zender-toggle="c20ad4d76fe97759aa27a0c99bff6710">Click to show modal</a>

HTML image example

<img src="image_link" zender-toggle="c20ad4d76fe97759aa27a0c99bff6710"/>

For widget blocks, clicking the copy button will give you this:

{block_c51ce410c124a10e0db5e4b97fc2af39}

You can then include this block code to any tpl files in the dashboard and default template folders. After adding the block code, navigate to the page and you will see the block widget show to the document dynamically.

Pages

The system supports dynamic pages, these page will work like any other normal internet page but navigating it inside zender is more intelligent.

After adding a page, you can easily copy the toggle attribute by clicking the copy button of the row. The page toggle attribute looks like this:

zender-page="2/435325"

To make the modal work, just add the toggle attribute to any links or images like this:

HTML link example

<a href="#" zender-page="2/435325">Click to show modal</a>

HTML image example

<img src="image_link" zender-page="2/435325"/>

Pages will work the same as any normal pages and can be accessed directly by link, the only difference is that it uses smart navigation when embedded inside zender.

Payments

These are the default payment gateways of zender, more gateways are already planned to be added in the next updates.

  • Paypal
  • Stripe
  • Mollie

The paypal payment gateway uses PayPal Pro, this is the latest version of paypal payment system. There is no need to integrate paypal IPN because the payments are validated directly after the user completes a transaction. This requires paypal business account.

We assume that you already have a paypal account, follow the instructions here to get your live paypal username, password and signature.

Paste the credentials in the system settings, to go live. You must disable paypal test mode.

If you just want to test the gate way using paypal sandbox, go here to get your test credentials then paste it in the systems setting, you will also need to enable paypal test mode.

For stripe, the setup is pretty simple and straightforward. We assume that you already have a stripe account.

Login to your stripe account, then copy and paste your stripe publishable key and secret key to zender system settings.

Emails

Zender supports sending emails using php's native mail() function or thru remote smtp server. To use remote smtp server, follow the steps below, we will use gmail smtp in this guide.

The input fields for remote smtp are automatically populated by default to gmail settings, you'll just need to add your gmail email and password then save the system settings.

To use gmail smtp, you will need to enable less secure apps here.

Recaptcha

To protect your site from spam and bots, it is required to set up your site's recaptcha.

Go to google developer console here

Click the + (plus) icon on the top right of the page.

Follow these steps:

  • Enter a label name, eg. MySite
  • Select reCaptcha v2 and "I'm not a robot" tickbox
  • Add your site's domain
  • Accept the reCaptcha Terms of Service
  • Click the Submit button
  • Now copy the site key and secret key then paste in in your zender system settings. You're done!

App Builder

Starting your own SaaS platform requires branding and with zender it's completely possible!

An integrated builder for the gateway app is included in system, we will do the dirty work for you. You'll be able to completely change the app's look and feel by adding your own package name, app icon, app logo, app splash screen and app colors.

The server gateway api is also integrated to the app itself, your users won't need to manually input the server url. They just need to scan their account qr code to register devices then start sending messages.

Before sending build request, make sure you did the following:

  • A unique package name, we recommend mapping your app's package name to your site domain. For example: com.mydomain.gateway

    Once you specified a package name, we recommend to not change it because the unique id's of the devices depends on this. Changing of package name can affect user accounts and registered devices! You have been warned!

  • Attractive app name, you can map the gateway app name to your site's brand name. For example: Zender

  • App color, we recommend that the app color you choose matches the app icon/logo and app splashcreen background colors.

    This color will be used for status bar and app main theme color.

  • Send interval, this is the interval between each message sending. Currently, this is hardcoded to the app itself. We already have plans to allow your customers to choose their preferred interval per device. Please read the official zender roadmap in the community forums. We recommend setting this value >= 5 seconds. Greater is better and saves a lot of compute time.

  • Receive interval, the amount of time before sending new received messages to server. The data contains all messages sent the whole day, so we recommend that this value be >= 60 seconds. Greater is better and saves a lot of compute time.

  • App icon image, the image must be in png format with fixed resolution size of 1024x1024 in px. Your main brand art must be centered. Please view the sample app images in the Builder directory included in your codecanyon downloaded zip file.

  • App logo image, the image must be in png format with fixed resolution size of 251x53 in px. This is the image shown as logo inside the gateway app. Please view the sample images in the Builder directory included in your codecanyon downloaded zip file.

  • App splashscreen image, the image must be in png format with fixed resolution size of 2732x2732 in px. Your main brand art must be centered. Please view the sample app images in the Builder directory included in your codecanyon downloaded zip file.

  • Builder email, this is important, we will send a message in this email when our system is done building your app. Your app should be automatically uploaded to your server after building if the folder permissions to uploads/builder/ was properly set to 775. When the app is uploaded to your server, it is immediately available for download in the "Add Device" form.

After completing the requirements metioned above, add your codecanyon purchase code in the system settings then click the "Build App" button in the admin panel.

Wait for our email message verfying the successful build!
